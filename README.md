# Danveer Singh — Personal Academic Website

A multi-page academic website built with HTML and CSS — designed to be hosted on GitHub Pages for free.

## Pages

- `index.html` — Home (Light @ Nanoworld)
- `research.html` — Research projects
- `people.html` — Lab members
- `publications.html` — Full publication list
- `cv.html` — Curriculum Vitae
- `gallery.html` — Photo gallery with filters and lightbox
- `contact.html` — Contact info & online profiles
- `styles.css` — Shared stylesheet

## How to Host on GitHub Pages (Free, Forever)

### Step 1 — Create a GitHub Account

1. Go to https://github.com
2. Click **Sign up**
3. Choose a username — **important:** use something professional like `danveersingh` (your final URL will be `danveersingh.github.io`)
4. Verify your email

### Step 2 — Create a New Repository

1. Click the **+** icon at the top-right and select **New repository**
2. Repository name: **`<your-username>.github.io`** (must match your username exactly, e.g., `danveersingh.github.io`)
3. Set it to **Public**
4. Check **Add a README file**
5. Click **Create repository**

### Step 3 — Upload These Files

1. On your new repository page, click **Add file → Upload files**
2. Drag all of these files in:
   - `index.html`
   - `research.html`
   - `people.html`
   - `publications.html`
   - `cv.html`
   - `gallery.html`
   - `contact.html`
   - `styles.css`
3. Scroll down, type a commit message like "Initial site upload"
4. Click **Commit changes**

### Step 4 — Wait & Visit

1. GitHub Pages auto-publishes within 1–2 minutes
2. Visit: **https://<your-username>.github.io**
3. Done! 🎉

### Step 5 — Update Google Sites

You can either delete your old Google Sites page or add a single banner saying "This site has moved to <new-url>" with the new link.

## Editing Content Later

To update any text, photos, or links:
1. Open the repository on GitHub
2. Click any `.html` file
3. Click the **pencil/edit** icon
4. Make changes and click **Commit changes**
5. The site updates within seconds

## Custom Domain (Optional)

If you ever buy a domain like `danveersingh.com`:
1. Add a file named `CNAME` to the repository with just `danveersingh.com` in it
2. Point your domain's DNS to GitHub Pages
